Implementing Least-Privilege Access Control in Azure Using RBAC

Overview

This project demonstrates the implementation of Role-Based Access Control (RBAC) in Microsoft Azure to enforce least-privilege access and strengthen cloud governance.

Business Problem

Organizations frequently grant excessive permissions to users, creating unnecessary security risk and increasing the attack surface. A structured access model was required to ensure users only received permissions necessary for their roles.

Objectives

  • Create role-specific security groups
  • Assign appropriate Azure RBAC roles
  • Validate access restrictions
  • Review governance and audit controls

Architecture

Azure Entra ID Groups → Azure RBAC Roles → Resource Group → Virtual Machines

Implementation

  1. Created Senior Admins, Junior Admins, and Service Desk groups
  2. Assigned Virtual Machine Contributor permissions
  3. Validated VM access capabilities
  4. Tested restricted administrative actions
  5. Reviewed audit logs and governance controls

Security Outcome

  • Least-privilege access enforced
  • Unauthorized actions prevented
  • Administrative activities auditable
  • Improved governance posture

Key Skills Demonstrated

  • Azure RBAC
  • Identity & Access Management
  • Zero Trust Principles
  • Cloud Governance
  • Security Auditing

Lessons Learned

RBAC is a foundational cloud security control that enables secure delegation of responsibilities while reducing operational risk.
